# Инструкция по развертыванию проекта Mioplan

## Обзор проекта

Проект Mioplan состоит из двух частей:
1. **Фронтенд** - React-приложение для визуализации задач
2. **Бэкенд** - Node.js/Express сервер для работы с данными

URL фронтенда: https://mioplan.vercel.app/
URL сервера: https://mioplan.onrender.com

## Подготовка к развертыванию

1. **Подготовка базы данных**
   - Создайте базу данных PostgreSQL (например, на Render, ElephantSQL или другом сервисе)
   - Запишите параметры подключения (имя базы данных, пользователя, пароль, хост, порт)

## Развертывание бэкенда на Render

1. **Подготовка репозитория**
   - Убедитесь, что в репозитории есть следующие файлы:
     - `server.js` - основной файл сервера
     - `package.json` - зависимости и скрипты
     - `Procfile` - инструкции для запуска на Render
     - `.env` - переменные окружения (при развертывании задайте их в настройках Render)

2. **Развертывание на Render**
   - Зарегистрируйтесь на [Render](https://render.com/)
   - Создайте новый "Web Service"
   - Подключите ваш GitHub репозиторий
   - Настройте параметры:
     - **Name**: mioplan-backend (или другое имя)
     - **Runtime**: Node
     - **Build Command**: `npm install`
     - **Start Command**: `npm start`
     - **Instance Type**: Free (для начала)
   - Добавьте переменные окружения:
     - `PORT`: `10000`
     - `NODE_ENV`: `production`
     - `DB_NAME`: `mioplan_db`
     - `DB_USER`: `mioplan_user`
     - `DB_PASSWORD`: `secure_password123`
     - `DB_HOST`: `aws-01.db.example.com`
     - `DB_PORT`: `5432`
   - Нажмите "Create Web Service"
   - Дождитесь завершения деплоя и запишите URL вашего бэкенда

3. **Проверка работы**
   - После развертывания откройте URL вашего сервиса
   - Добавьте `/` в конец URL (например, `https://mioplan-backend.onrender.com/`)
   - Вы должны увидеть сообщение: `{"message":"Mioplan API server is running"}`

## Развертывание фронтенда на Vercel

1. **Подготовка**
   - Убедитесь, что в корне проекта есть файл `vercel.json`
   - Убедитесь, что папка `client` содержит полный фронтенд проект

2. **Развертывание на Vercel**
   - Зайдите на [vercel.com](https://vercel.com) и войдите в свою учетную запись
   - Создайте новый проект ("New Project")
   - Импортируйте ваш GitHub репозиторий
   - Настройте проект:
     - **Project Name**: mioplan-frontend (или другое имя)
     - **Framework Preset**: Vite
     - **Root Directory**: client
   - Добавьте переменные окружения при необходимости
   - Нажмите "Deploy"
   - Дождитесь завершения деплоя

3. **Проверка работы**
   - После развертывания откройте URL вашего сайта
   - Убедитесь, что приложение загружается и отображает матрицу задач

## Интеграция фронтенда и бэкенда

После того как обе части развернуты:

1. Обновите конфигурацию прокси в `client/vite.config.js`, чтобы она указывала на URL вашего сервера на Render:
   ```javascript
   server: {
     proxy: {
       '/api': {
         target: 'https://mioplan.onrender.com',
         changeOrigin: true,
         secure: true
       }
     }
   }
   ```

2. Обновите CORS настройки в `server/server.js`, добавив туда URL вашего фронтенда на Vercel:
   ```javascript
   app.use(cors({
     origin: [
       'http://localhost:5176', 
       'https://mioplan.vercel.app', 
       /\.vercel\.app$/, 
       'https://mioplan-frontend.vercel.app'
     ],
     credentials: true,
     methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
     allowedHeaders: ['Content-Type', 'Authorization'],
     exposedHeaders: ['Content-Range', 'X-Content-Range']
   }));
   ```

3. Передеплойте обе части приложения

## Возможные проблемы и их решение

### Проблема: "Сервер недоступен. Пожалуйста, попробуйте позже."
- **Причина**: Бэкенд не развернут или недоступен
- **Решение**: 
  1. Проверьте, что бэкенд развернут на Render и работает
  2. Убедитесь, что в настройках Vercel указана правильная переменная `REACT_APP_API_URL`
  3. Проверьте логи бэкенда на Render

### Проблема: "Ошибка загрузки задач: Request failed with status code 404"
- **Причина**: Неверный URL или ID доски
- **Решение**:
  1. Проверьте правильность настроек API в приложении
  2. Убедитесь, что ID доски существует и вы имеете к нему доступ
  3. Проверьте правильность API ключа

### Проблема: CORS ошибки в браузере
- **Причина**: Неверные настройки CORS на бэкенде
- **Решение**:
  1. Убедитесь, что в бэкенде установлены правильные настройки CORS
  2. Проверьте, что в Vercel и Render нет ограничений по кросс-доменным запросам
