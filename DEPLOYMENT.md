# Инструкция по развертыванию проекта Mioplan

## Обзор проекта

Проект Mioplan состоит из двух частей:
1. **Фронтенд** - React-приложение для визуализации задач Kaiten
2. **Бэкенд** - Node.js/Express сервер для работы с данными и проксирования запросов к Kaiten API

## Развертывание бэкенда на Render

1. **Подготовка репозитория**
   - Скопируйте папку `server` в отдельный репозиторий на GitHub
   - Убедитесь, что в репозитории есть следующие файлы:
     - `server.js` - основной файл сервера
     - `package.json` - зависимости и скрипты
     - `.env` - переменные окружения
     - `Procfile` - инструкции для запуска на Render

2. **Развертывание на Render**
   - Зарегистрируйтесь на [Render](https://render.com/)
   - Создайте новый "Web Service"
   - Подключите ваш GitHub репозиторий
   - Настройте параметры:
     - **Name**: mioplan-backend (или другое имя)
     - **Runtime**: Node
     - **Build Command**: `npm install`
     - **Start Command**: `npm start`
     - **Instance Type**: Free (для начала)
   - Добавьте переменные окружения:
     - `PORT`: `10000`
     - `NODE_ENV`: `production`
   - Нажмите "Create Web Service"

3. **Проверка работы**
   - После развертывания откройте URL вашего сервиса
   - Добавьте `/` в конец URL (например, `https://mioplan-backend.onrender.com/`)
   - Вы должны увидеть сообщение: `{"message":"Mioplan API server is running"}`

## Развертывание фронтенда на Vercel

1. **Подготовка репозитория**
   - Убедитесь, что фронтенд код находится в репозитории на GitHub
   - Проверьте наличие файла `package.json` с необходимыми скриптами

2. **Развертывание на Vercel**
   - Зарегистрируйтесь на [Vercel](https://vercel.com/)
   - Импортируйте ваш GitHub репозиторий
   - Настройте параметры:
     - **Framework Preset**: React
     - **Build Command**: `npm run build`
     - **Output Directory**: `dist` (или другая папка сборки)
   - Добавьте переменные окружения:
     - `REACT_APP_API_URL`: URL вашего бэкенда (например, `https://mioplan-backend.onrender.com/api`)
   - Нажмите "Deploy"

3. **Проверка работы**
   - После развертывания откройте URL вашего сайта
   - Убедитесь, что приложение загружается и отображает матрицу задач

## Настройка подключения к Kaiten API

1. **Получение API ключа**
   - Войдите в свою учетную запись Kaiten
   - Перейдите в настройки профиля
   - Выберите раздел "API-ключи"
   - Создайте новый ключ с необходимыми правами доступа
   - Скопируйте полученный ключ

2. **Настройка в приложении**
   - Откройте ваше приложение на Vercel
   - Нажмите кнопку "Настройки API" (⚙️)
   - Заполните поля:
     - **URL-адрес Kaiten**: ваш URL Kaiten (например, `https://yourcompany.kaiten.ru`)
     - **API ключ**: полученный на предыдущем шаге
     - **ID доски**: ID доски, с которой нужно получить задачи
   - Нажмите "Сохранить"

3. **Проверка работы**
   - После сохранения настроек приложение автоматически загрузит задачи с указанной доски
   - Задачи должны отобразиться в соответствующих ячейках матрицы

## Возможные проблемы и их решение

### Проблема: "Сервер недоступен. Пожалуйста, попробуйте позже."
- **Причина**: Бэкенд не развернут или недоступен
- **Решение**: 
  1. Проверьте, что бэкенд развернут на Render и работает
  2. Убедитесь, что в настройках Vercel указана правильная переменная `REACT_APP_API_URL`
  3. Проверьте логи бэкенда на Render

### Проблема: "Ошибка загрузки задач: Request failed with status code 404"
- **Причина**: Неверный URL или ID доски Kaiten
- **Решение**:
  1. Проверьте правильность URL Kaiten в настройках
  2. Убедитесь, что ID доски существует и вы имеете к нему доступ
  3. Проверьте правильность API ключа

### Проблема: CORS ошибки в браузере
- **Причина**: Неверные настройки CORS на бэкенде
- **Решение**:
  1. Убедитесь, что в бэкенде установлено `origin: '*'` в настройках CORS
  2. Проверьте, что в Vercel и Render нет ограничений по кросс-доменным запросам
