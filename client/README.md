# Kaiten Matrix

React-приложение для отображения задач из Kaiten API на интерактивной матрице важности и сложности.

## Описание проекта

Приложение позволяет визуализировать задачи из Kaiten в виде матрицы 3x3, где:
- Горизонтальные оси матрицы — Сложность: Сложно, Средне, Просто (слева направо).
- Вертикальные оси — Важность: Важно, Средне, Минимум (сверху вниз).

Каждая задача отображается как отдельная карточка с названием, описанием и метками в соответствующей ячейке матрицы.

## Технологии

- React 18
- Vite
- Axios
- CSS

## Установка и запуск

### Требования

Для работы приложения необходимо:
- Node.js (версии 16 или выше)
- NPM или Yarn

### Установка зависимостей

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/kaiten-matrix.git
cd kaiten-matrix
```

2. Установите зависимости:
```bash
npm install
```

### Запуск приложения

Для запуска приложения в режиме разработки выполните:
```bash
npm run dev
```

Приложение будет доступно по адресу: `http://localhost:5173`

### Сборка для продакшена

Для создания оптимизированной сборки выполните:
```bash
npm run build
```

Для предварительного просмотра продакшен-сборки локально:
```bash
npm run preview
```

## Настройка подключения к Kaiten API

### Получение API ключа

1. Войдите в свою учетную запись Kaiten.
2. Перейдите в настройки профиля.
3. Выберите раздел "API-ключи".
4. Создайте новый ключ с необходимыми правами доступа.
5. Скопируйте полученный ключ.

### Настройка в приложении

1. В интерфейсе приложения нажмите кнопку "Настройки API".
2. Заполните поля:
   - URL-адрес Kaiten (например, `https://yourcompany.kaiten.ru`)
   - API ключ (полученный на предыдущем шаге)
   - ID доски, с которой нужно получить задачи
3. Нажмите "Сохранить".

После сохранения настроек приложение автоматически загрузит задачи с указанной доски.

## Работа с приложением

### Обновление задач

Для обновления списка задач нажмите кнопку "Обновить" в заголовке приложения.

### Матрица задач

Задачи автоматически распределяются по матрице в зависимости от их важности и сложности:

- **Важно / Сложно**: задачи с высоким приоритетом и высокой сложностью
- **Важно / Средне**: задачи с высоким приоритетом и средней сложностью
- **Важно / Просто**: задачи с высоким приоритетом и низкой сложностью
- **Средне / Сложно**: задачи со средним приоритетом и высокой сложностью
- **Средне / Средне**: задачи со средним приоритетом и средней сложностью
- **Средне / Просто**: задачи со средним приоритетом и низкой сложностью
- **Минимум / Сложно**: задачи с низким приоритетом и высокой сложностью
- **Минимум / Средне**: задачи с низким приоритетом и средней сложностью
- **Минимум / Просто**: задачи с низким приоритетом и низкой сложностью

### Таймлайн задач

Приложение также предоставляет представление задач в виде таймлайна, где можно:

1. **Переключение режимов**: Используйте переключатель "Таймлайн" для включения/выключения отображения таймлайна.

2. **Визуализация секторов**: Карточки задач наследуют информацию о своем секторе (важность/сложность) и отображаются с соответствующими цветами и подписями.

3. **Drag-and-drop**: Перетаскивайте карточки задач на таймлайне и растягивайте их по горизонтали для задания или изменения сроков выполнения.

4. **Структура таймлайна**: Таймлайн разбит на месяцы, каждый месяц - на недели, с четкой визуальной сеткой.

5. **Удаление задач**: Удаляйте задачи с таймлайна с помощью кнопки удаления на каждой карточке.

6. **Фильтрация и поиск**: Используйте панель управления таймлайном для поиска задач по названию и фильтрации по секторам.

7. **Массовое выделение**: Используйте функцию массового выделения для одновременного перемещения нескольких задач.

8. **Уведомления о пересечениях**: Получайте уведомления, если задачи пересекаются по срокам в одном секторе.

9. **Критические задачи**: Автоматически получайте визуальные индикаторы для задач с истекающими сроками (осталось 3 дня или менее).

10. **Экспорт**: Экспортируйте таймлайн в PDF/PNG для совместного использования с командой.

11. **История изменений**: Используйте функции Undo/Redo для отмены и повтора действий с карточками на таймлайне.

Все изменения (перемещение, изменение срока, удаление) сохраняются в состоянии приложения и, по возможности, синхронизируются с Kaiten.
